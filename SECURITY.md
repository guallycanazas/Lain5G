# Security Policy

## Release status

`1.0.0-rc.1` is an unreleased release candidate. No final release, guaranteed
support period, or production suitability is claimed. Security fixes are
handled on a best-effort basis for the current development line.

## Reporting a vulnerability

Do not disclose exploit details, credentials, subscriber data, private network
information, or RF authorization records in a public issue.

## Report contents

When a private channel is available, include the affected version or commit,
impact, prerequisites, and a minimal sanitized reproduction. State whether the
issue affects the application, deployment isolation, secrets handling, or an
upstream dependency. Do not start services, access hardware, or transmit RF
solely to demonstrate a report.

For third-party components, also follow the upstream project's disclosure
process. General support requests belong in [SUPPORT.md](SUPPORT.md).
